# Backup & Restore Runbook (SD2-26)

## 1) 목표
- 대상: 학생 정보, 성적, 피드백, 상담, 알림, 감사/보안 로그
- RPO: 24시간 이내
- RTO: 2시간 이내

## 2) 백업 정책
- 기본: MongoDB Atlas Cloud Backup 사용
- 주기:
  - 증분 백업: 6시간 간격
  - 스냅샷 보관: 14일
  - 월간 스냅샷: 6개월
- 접근 권한:
  - 운영 관리자(교사 역할 아님) 최소 인원만 Atlas backup restore 권한 보유
  - 프로젝트 읽기 권한과 복구 권한 분리

## 3) 복구 절차
1. 장애 범위 확인 (단일 컬렉션 / 전체 DB)
2. 복구 시점 결정 (장애 직전 정상 시각)
3. Atlas PITR(Point-in-Time Restore)로 임시 클러스터 복구
4. 검증 항목 수행:
   - 학생 수/성적 건수/상담 건수 샘플 비교
   - 최근 24시간 C/U/D 감사 로그 존재 확인
   - 보안 이벤트 로그 존재 확인
5. 서비스 전환:
   - 임시 클러스터를 운영 연결 대상으로 승격하거나
   - 운영 클러스터로 데이터 재적재 후 애플리케이션 재기동

## 4) 정합성 점검 체크리스트

복구 완료 후 아래 스크립트를 실행한다:

```bash
MONGODB_URI="<복구된_클러스터_URI>" npx tsx scripts/verify-integrity.ts
```

스크립트가 자동으로 검증하는 항목:
- 학생-학부모 연결(`parent_student`) 무결성 — 고아 참조 탐지
- 성적(`grade.calculated_grade`) 등급 재계산 일치 여부
- OLAP 스냅샷(`student_learning_snapshot`) 평균점수 드리프트 (10% 초과 시 경고)
- 최근 24시간 AuditLog 연속성

오류 건수 > 0이면 exit code 1로 종료되며, 서비스 전환 전 반드시 수동 확인이 필요하다.

## 5) 월간 복구 리허설
- 매월 1회 샘플 학생 3명 기준으로 복구 리허설 수행
- 결과 기록:
  - 복구 시작/완료 시각
  - 실제 RTO
  - 불일치 데이터 및 후속 조치
